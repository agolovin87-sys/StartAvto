# Деплой, сохранение в GitHub и откат

Команды выполняются из папки проекта:

```powershell
cd c:\Users\start\Desktop\StartAvto
```

---

## Одна команда: деплой + сохранение точки отката

Собирает сайт, публикует на **Firebase Hosting**, делает **коммит** и **тег** `deploy-…` в Git, отправляет всё на **GitHub**.

### Вариант 1 (короткий)

```powershell
npm run deploy
```

### Вариант 2 («спаси и сохрани»)

```powershell
npm run "спаси и сохрани"
```

Обе команды делают одно и то же.

### Что попадает в сохранение

- В **коммите** и **теге** указываются **дата и время** (локально и UTC) и **список изменённых файлов** (если они были).
- Если кода не меняли и только перепубликовали сайт — создаётся **пустой коммит** с пояснением; тег всё равно ставится.

### Просмотр истории деплоев

```powershell
git log --oneline --grep="^deploy:" -20
git tag -l "deploy-*"
git show ИМЯ_ТЕГА
```

Пример: `git show deploy-20260410-153045-123`

## Откат к сохранённой точке

Интерактивно показывает список точек (тегов `deploy-*`) и после выбора выполняет **`git reset --hard`** к выбранному тегу.

```powershell
npm run откат
```

1. Выводится нумерованный список (от новых к старым).
2. Введите **номер** нужной точки или **0** для отмены.
3. Если есть несохранённые правки в папке — скрипт предупредит.
4. Подтвердите откат словом **да**.

**Важно:**

- Откатывается **только локальный репозиторий**. Сайт на Firebase **не** откатывается автоматически.
- Чтобы **опубликовать** откат на хостинг, после отката снова выполните:

  ```powershell
  npm run "спаси и сохрани"
  ```

- Чтобы **перезаписать** ветку на GitHub (если откатились назад и хотите, чтобы на сервере было то же самое):

  ```powershell
  git push --force-with-lease origin main
  ```

  Используйте осторожно, если с репозиторием работают другие люди.

## Другие полезные команды

| Действие | Команда |
|----------|---------|
| Локальная разработка | `npm run dev` |
| Только сборка (без деплоя) | `npm run build` |
| Просмотр собранного сайта локально | `npm run preview` |
| Вход в Firebase CLI (если просит) | `npx firebase-tools@latest login` |

---

## Требования

- Установлены **Node.js** и **Git**.
- Настроен **Firebase** (файлы `.firebaserc`, `firebase.json`) и выполнен вход: `npx firebase-tools@latest login`.
- Настроен **remote** на GitHub (`git remote -v`).

---

## Откат только сайта на Firebase (без Git)

В [консоли Firebase → Hosting](https://console.firebase.google.com) можно выбрать предыдущий релиз и откатить хостинг, не трогая Git.
